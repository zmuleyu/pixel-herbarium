## プロジェクト情報ニーズ分析

検索戦略を策定する前に、本プロジェクトが回答すべきコア問題を明確にする。

| 調査軸 | コア問題 | 対応機能 |
|--------|---------|---------|
| **撮影習慣** | 屋外で植物・花を撮影する際のユーザー行動パターンは？ | Discover Tab（カメラコアフロー） |
| **収集動機** | 日本ユーザーがスタンプ・コレクション系アプリを継続使用する動機は？ | Herbarium 60格図鑑 |
| **地図プライバシー** | 位置共有に対するユーザーの許容範囲はどこか？ | City Map ±100m ファジング |
| **シェアチャネル** | LINE・Instagram・Twitter それぞれに適したコンテンツ形式は？ | ポスター生成 + ShareSheet |
| **サブスク受容度** | 月額制 vs 買い切りに対する日本ユーザーの選好は？ | Apple IAP クォータシステム |
| **季節行動** | 花見期間中のアプリ使用頻度とシーンはどう変化するか？ | 桜まつり 季節イベントシステム |
| **UI痛点** | カメラ系・図鑑系アプリで最も多い操作上の不満は？ | 全体UXデザイン参考 |

## PART 1｜日本本土ユーザーノートプラットフォーム

### 1. Note.com — 生活シーンと使用日記

**プラットフォーム定義：** 「花見シーン」での自然な行動と、コレクション系アプリへの感情表現を掘り起こす。

**検索入口：** https://note.com/search

#### 検索ワードマトリクス

```
# 撮影・植物シーン
アプリ 習慣              → 基本習慣日記
花 写真 アプリ           → 花撮影アプリ使用体験
お花見 スマホ            → 花見期間のスマホ行動
植物 図鑑 アプリ         → 植物図鑑系アプリ体験

# 収集・ゲーミフィケーション
コレクション アプリ 続ける → コレクションアプリを続ける理由
スタンプ ラリー アプリ    → スタンプラリー系アプリの行動
ガチャ 課金 やめた        → 課金メカニズムのネガティブ体験（地雷回避）

# シェア行動
LINE 写真 シェア          → LINEシェア習慣
Instagram 投稿 日常       → INS投稿動機
アプリ 友達 共有          → アプリ内ソーシャルシェア

# サブスク・課金
サブスク アプリ 解約      → 解約トリガーとなった原因
月額 アプリ 続ける理由    → サブスクを継続する理由
```

**重点収集シグナル：**

- 「やめた理由」の言及 → 離脱トリガーポイント
- 「毎日使ってる」の言及 → リテンションフック
- 「友達に教えた」の言及 → ソーシャル拡散ノード
- 「位置情報 怖い」の言及 → プライバシー懸念の境界

**推奨フィルタ条件：** 期間は 2024年以降に絞る。スキ数 50 以上の記事を優先（共感度が高い証拠）。


### 3. Applion — 競合レビュー・痛点抽出

**プラットフォーム定義：** 類似アプリのユーザー評価を直接入手し、Pixel Herbarium の機能設計にマッピングする。

**検索入口：** https://applion.jp/

#### 競合アプリ検索リスト

| 競合タイプ | 検索ワード | 調査目的 |
|-----------|-----------|---------|
| 植物識別系 | `Picturethis レビュー` | 識別速度・精度に対するユーザーの許容下限 |
| 植物識別系 | `GreenSnap レビュー` | 日本ローカルコミュニティシェア機能の評価 |
| ピクセル・レトロ系 | `ピクセルアート アプリ レビュー` | ピクセルアートスタイルの受容度 |
| 図鑑収集系 | `図鑑 コレクション アプリ レビュー` | 60格図鑑フォーマットへの反応 |
| マップソーシャル系 | `マップ 位置情報 アプリ 怖い` | 位置プライバシーへのネガティブ評価 |

#### 高価値レビュー抽出ラベル

```
【ポジティブシグナル → 機能維持・強化】
「毎日開いてる」「コンプリートしたい」「友達に自慢」
「かわいい」「センスがいい」「続けられる」

【ネガティブシグナル → 回避すべき設計】
「毎回確認が面倒」「重い」「通知がうるさい」
「位置情報 なんで必要」「課金しないと遊べない」「友達いないと意味ない」

【サブスク関連シグナル】
「高い」「月額 やめた」「一回払いにして」「無料でも使える」
```

### 4. App Ape（アップエイプ）

**プラットフォーム定義：** 日本市場の定量基準データを取得し、Pixel Herbarium の目標ユーザー行動仮説を検証する。

#### 取得すべきコア指標

```markdown
## 目標データポイントリスト

### セッション・使用頻度
- [ ] 植物識別系アプリの1日平均起動回数
- [ ] カメラ・撮影系アプリの平均1回あたり使用時間
      （「<10秒AIフロー」設計目標との比較）
- [ ] 図鑑・コレクションアプリの30日リテンション率ベンチマーク

### ユーザー属性
- [ ] 「花 写真」「散歩」関連アプリの性別・年齢分布
      → 「20〜30代女性」がコアユーザーかを検証
- [ ] 屋外・自然系アプリの地域分布（関東圏占有率）

### 季節変動
- [ ] 植物・自然系アプリの3〜5月ダウンロード数変動係数
      → 「桜シーズン」のトラフィックボーナスを定量化

### 競合データ
- [ ] GreenSnap・PictureThis JP の MAU トレンド
- [ ] 同類アプリの平均アンインストールサイクル
```

**検索戦略：**

1. 「無料レポート」セクションで「ライフスタイル」「写真・カメラ」カテゴリのレポートを優先ダウンロード
2. サイト内検索：`植物` `図鑑` `カメラ` で専門データを検索
3. 「季節性トレンド」レポートで3月下旬〜4月のユーザーアクティビティピークを確認

### LINE プラットフォームの制約と補完戦略

#### プラットフォーム固有の制限

| 制限事項 | 原因 | 対応策 |
|---------|------|-------|
| プライベートトークは不可視 | エンドツーエンド暗号化 | OpenChat の公開チャンネルに転換 |
| UGC 向け公開 API なし | LINE ポリシー制限 | 手動観察・記録のみ（自動化不可） |
| BLOG コンテンツが少ない | ユーザーが INS/Note に移行中 | LINE BLOG はサブ情報源として扱う |
| OpenChat の検索結果が限定的 | アルゴリズムが不透明 | 複数キーワードを組み合わせ、各ワードを独立して検索 |
