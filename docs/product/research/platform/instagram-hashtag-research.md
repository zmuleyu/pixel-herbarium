# 🌸 Instagram 季节性自然内容检索方案
## —— 以赏花 / 红叶场景为核心的用户行为研究框架

> **目标**：通过 Instagram hashtag 矩阵，系统性挖掘日本用户在赏樱 / 红叶场景下的真实行为、工具偏好与内容习惯，为产品设计与用户研究提供数据支撑。

---

## 一、核心 Hashtag 检索矩阵

Hashtag 按"流量规模 × 用户意图深度"分为三级，建议从一级泛标签切入定性方向，再用二、三级标签精准定位目标用户群。

### 🔴 一级标签 —— 泛流量入口（帖子量级：百万+）

| 标签 | 语言 | 场景说明 |
|------|------|----------|
| `#桜` | 日文 | 樱花核心词，覆盖最广 |
| `#お花見` | 日文 | 赏花行为，含聚餐/野餐场景 |
| `#紅葉` | 日文 | 红叶核心词 |
| `#紅葉狩り` | 日文 | 红叶观赏行为（"狩り"含主动寻访意味） |
| `#sakura` | 英文 | 面向外国游客或双语用户 |
| `#autumnleaves` | 英文 | 秋叶泛流量，含非日本内容，需过滤 |

> **使用策略**：一级标签适合做**宏观趋势扫描**，不建议直接用于用户访谈对象筛选，信噪比较低。

---

### 🟡 二级标签 —— 地域 / 场景信息层（帖子量级：万~十万）

| 标签 | 信息维度 | 研究价值 |
|------|----------|----------|
| `#桜2024` `#紅葉2024` | 时间锚点 | 追踪特定年份的集中爆发期 |
| `#満開` | 物候状态 | 用户对"最佳观赏时机"的主动判断 |
| `#桜前線` | 气象/迁移路径 | 了解用户是否主动追踪开花进度 |
| `#紅葉スポット` | 地点信息 | 发现高频被提及的具体地点 |
| `#花見弁当` | 配套行为 | 赏花与饮食准备的关联场景 |

> **使用策略**：二级标签是**场景细分**的核心层，可有效区分"路过顺拍派"与"主动规划派"用户。

---

### 🟢 三级标签 —— 深度用户行为标识（帖子量级：千~万）

| 标签 | 行为特征 | 目标用户画像 |
|------|----------|--------------|
| `#桜パトロール` | 定期巡查记录 | 高频、本地化、习惯性记录者 |
| `#桜記録` | 个人档案意识 | 有长期记录意愿，重视时间轴 |
| `#桜コレクション` | 收藏/比较心态 | 对品种或地点有系统性兴趣 |
| `#紅葉巡り` | 多地点巡游行为 | 主动规划路线，有探索欲 |
| `#花日記` | 日记式记录 | 高情感投入，内容连续性强 |
| `#植物のある暮らし` | 生活方式标签 | 与植物/自然深度绑定的生活态度 |
| `#お花見コーデ` | 着装/穿搭场景 | 赏花的仪式感与社交展示需求 |
| `#桜ポートレート` | 摄影构图意识 | 技术驱动型用户，设备偏好可挖 |

> **使用策略**：三级标签用户是**定性研究的黄金访谈对象**，内容深度高、行为动机明确。

## 二、分析操作步骤（SOP）

### Step 1：标签搜索 —— 切换至"最近"视图

```
搜索入口：Instagram 搜索栏 → 输入 hashtag → 切换至【最近】Tab（而非【トップ】）
```

- **原因**："トップ"（热门）展示的是算法加权的高赞内容，存在创作者/网红污染
- **"最近"** 展示的是时间顺序的真实用户发帖，更能反映普通用户行为
- **建议采样**：每个标签浏览 30~50 条，记录发帖频率、时间分布与内容模式

---

### Step 2：帖子结构分析

对每条帖子进行以下维度的结构化拆解：

#### 2-1. Caption 行动描述
- 识别关键句式：`「〇〇公園に行った」` / `「今年も来れた」` / `「初めて〇〇で見た」`
- 关注：**是独自行动还是集体行动**（「一人で」vs「家族で」vs「友達と」）
- 关注：**是否提及提前规划**（「調べて行った」「予報見て行った」）

#### 2-2. App 截图 / 引用工具识别

常见第三方工具截图（高度说明用户在赏花场景使用的数字工具）：

| 工具名 | 类型 | 说明 |
|--------|------|------|
| ウェザーニュース | 天气 App | 日本主流天气/物候预报，含"桜开花预报"功能 |
| GreenSnap | 植物识别 App | 用户上传花卉照片做识别，常见于植物爱好者 |
| Yahoo!天気 | 天气 App | 大众型天气工具 |
| 花見特集（各地方サイト） | 网页截图 | 用户在帖子中引用地方旅游网站的赏花路线 |

#### 2-3. 时间规律分析
- 记录发帖的**月份**（桜：3~4月；紅葉：10~11月）
- 记录**发帖时间段**（傍晚/周末集中，说明赏花是下班/休假行为）
- 结合 Caption 判断：**是即时发帖还是事后整理发**

#### 2-4. 定位信息
- 是否带 Location Tag（📍）
- 定位精度：**市区级**（東京都）vs **公园级**（新宿御苑）vs **无定位**
- 有无附近公共交通提示（「〇〇駅から徒歩5分」）

---

### Step 3：账号 Bio 分析

点击高互动**普通用户**主页（避免选择粉丝数 >1万的账号），重点分析：

| Bio 元素 | 洞察维度 |
|----------|----------|
| 职业身份关键词 | 上班族 / 主婦 / カメラマン / 学生 |
| 地域信息 | 关东 / 关西 / 地方城市 |
| 兴趣标识 | 植物系 / 写真系 / グルメ系 |
| 发帖频率 & 风格 | 记录派（日常随拍）vs 创作派（构图精细） |
| 关注者/关注数比例 | 判断是否为真实普通用户 |

> **筛选建议**：优先选择粉丝数 200~2000 的账号，互动率高且内容真实。

---

### Step 4：Story Highlights 分析

Story Highlights 是用户对**长期兴趣的主动归档**，是研究习惯深度的重要信号：

- 是否专门建立「桜」「紅葉」「花日記」等 Highlight 分类
- 多年度记录（「2022桜」「2023桜」「2024桜」）→ **年度仪式感强烈**
- Highlight 封面设计精细度 → 判断用户对内容管理的重视程度
- 是否包含 Stories 中的"当时直播"内容（location sticker、poll、温度计等）

## 三、工具辅助方案

### 🛠 工具对比速览

| 工具 | 类型 | 适用场景 | 技术要求 | 费用 |
|------|------|----------|----------|------|
| **Instaloader** | Python CLI | 批量下载帖子 metadata，无需登录 | 中（需 Python） | 免费/开源 |
| **Apify Instagram Scraper** | SaaS | 抓取指定 hashtag 下帖子结构化数据 | 低（GUI操作） | 按量付费 |
| **Phantombuster** | 自动化平台 | Hashtag 自动化分析 + 批量抓取 | 低 | 有免费额度 |

---

### Instaloader（推荐：轻量级 metadata 抓取）

```bash
# 安装
pip install instaloader

# 抓取指定 hashtag 的帖子（不下载图片，仅获取 metadata）
instaloader "#桜パトロール" --no-pictures --no-videos --no-video-thumbnails

# 输出内容包括：
# - 发帖时间戳
# - Caption 文本
# - 点赞数 / 评论数
# - 用户名
# - Location ID（如有）
```

> ⚠️ 注意：Instagram 对未登录抓取有频率限制，建议每次抓取量不超过 200 条，并设置请求间隔。

---

### Apify Instagram Scraper（推荐：结构化大批量抓取）

- 访问：[apify.com/apify/instagram-scraper](https://apify.com/apify/instagram-scraper)
- 输入 hashtag 列表（支持批量）
- 输出字段：`shortCode` / `caption` / `hashtags` / `timestamp` / `locationName` / `likesCount` / `commentsCount` / `ownerUsername`
- 建议导出为 JSON，再用 Python/Excel 做二次筛选

---

### Phantombuster（推荐：自动化周期监控）

- 使用 **Instagram Hashtag Collector** phantom
- 可设定定时任务（如每周一抓取上周新增帖子）
- 适合追踪**季节性峰值**（3月底满开期 / 11月紅葉ピーク）的内容爆发节奏

## 四、数据整理与分析框架

抓取数据后，建议按以下维度建立分析表：

| 帖子ID | 发帖日期 | 标签层级 | 使用工具截图 | 是否带定位 | 行动描述类型 | 账号类型 | 备注 |
|--------|----------|----------|--------------|------------|--------------|----------|------|
| ... | ... | 一/二/三级 | 是/否（工具名） | 是/否（精度） | 即时/回顾/规划 | A/B/C/D | |

### 账号类型分类参考

| 类型 | 特征描述 |
|------|----------|
| **A 类 —— 仪式记录型** | 每年固定发帖，有 Highlight 归档，Bio 含植物/季节关键词 |
| **B 类 —— 随机分享型** | 偶发性发帖，Caption 简短，无系统性标签使用 |
| **C 类 —— 摄影创作型** | 构图精细，使用 `#桜ポートレート` 等摄影标签，设备信息明确 |
| **D 类 —— 社交聚合型** | 带 `#お花見コーデ`、聚会照片，关注点在人而非植物 |

## 五、执行时间节点建议

| 时段 | 操作重点 |
|------|----------|
| **开花预报期**（2月下旬~3月初） | 监控 `#桜前線` `#開花予想`，捕捉提前规划行为 |
| **満開ピーク期**（3月下旬~4月上旬） | 高频抓取一级标签，分析实时发帖节奏与峰值 |
