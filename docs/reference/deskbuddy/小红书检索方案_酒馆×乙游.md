# 小红书「SillyTavern × 乙游」内容检索与分析方案

## 一、研究背景与目标

本方案旨在系统性地检索和分析小红书平台上 SillyTavern（"酒馆"）与乙游交叉领域的用户生成内容，回答以下核心问题：

1. 乙游玩家在多大程度上已经接触或使用酒馆类工具？
2. 她们的使用动机、方式和痛点是什么？
3. 这一交叉社区的规模、活跃度和发展阶段如何？
4. 从传统乙游到 AI 角色扮演的"迁移路径"是怎样的？

**为什么选择小红书：** 小红书是中国女性用户占比最高的内容社区之一，与乙游玩家的人群画像高度重合。同时，小红书兼具公开可检索性和社区私密性（评论区深度讨论、收藏行为），使其成为观察这一半隐蔽生态的理想窗口。

## 二、检索关键词体系

## 三、检索执行流程

### 阶段一：广撒网（第1-3天）

**目标：** 建立内容全景图，估算各象限的内容体量。

**操作步骤：**

1. 使用一个全新小红书账号（冷启动视角），逐一输入 A 象限全部关键词，记录每个词的搜索结果数量、前20条笔记的基本信息
2. 同一账号继续检索 B、C、D 象限的核心关键词（每象限选5-8个高优先级词）
3. 对每个关键词，同时查看「综合」「最新」「最热」三个排序维度的结果差异
4. 记录搜索联想词（小红书搜索框的自动补全建议），这些联想词反映了平台对内容关联性的理解

**产出物：** 关键词-结果量映射表，初步判断哪些象限有足够内容支撑深度分析。

### 阶段二：定向深挖（第4-7天）

**目标：** 对有效象限进行内容抓取和分类。

**操作步骤：**

1. 对阶段一中有效果的关键词，逐条查看前50条笔记（或全部结果，取数量较少者）
2. 对每条相关笔记建立记录卡片（见下方数据采集模板）
3. 进入高相关性笔记的作者主页，浏览其他笔记，判断是否为"交叉用户"（同时发布乙游和酒馆内容）
4. 逐条阅读高相关笔记的评论区（通常评论区的信息密度高于正文）
5. 记录笔记末尾或评论区中提到的QQ群号、微信群、Discord链接等私域入口

**产出物：** 内容数据库（结构化记录表），关键用户列表，私域社区线索。

### 阶段三：算法验证（第8-10天）

**目标：** 验证小红书推荐算法是否将酒馆内容推送给乙游用户。

**操作步骤：**

1. 用一个"纯乙游"账号（浏览历史全部为乙游内容，无任何AI/酒馆浏览记录）执行以下操作：
   - 搜索酒馆相关关键词，观察是否出现乙游关联推荐
   - 不主动搜索，仅浏览首页推荐流，观察是否自然出现酒馆内容
   - 浏览一篇酒馆教程后，观察后续推荐流的变化
2. 用一个"纯酒馆"账号反向验证：浏览酒馆教程后，是否被推荐乙游内容
3. 记录推荐算法的反应时间和内容关联强度

**产出物：** 算法关联性报告，评估两个圈层在平台层面的连接程度。

## 五、分析框架

### 5.1 用户迁移路径分析

从数据中还原乙游玩家接触和使用酒馆的典型路径：

```
触发事件                 信息获取              工具选择              使用深度
（停服/氪金疲劳/       →（B站教程/小红书    →（云酒馆/本地部署   →（尝鲜/偶尔使用/
  塌房/剧情不满）         推荐/朋友安利）       /ChatGPT直接用）     深度依赖/放弃）
```

**操作方法：** 在评论区和正文中寻找叙事线索，如"因为XX游戏停服所以开始用酒馆"、"之前一直玩XX，现在主要用AI"等自述性表达。将这些叙事编码归类，统计各路径的频次。

### 5.2 内容生命周期分析

判断这一交叉社区处于哪个发展阶段：

| 阶段 | 特征信号 | 判断依据 |
|---|---|---|
| 萌芽期 | 零星的个人体验分享，评论区多为"这是什么" | 笔记量 <50，以D象限为主 |
| 成长期 | 系统化教程出现，角色卡分享活跃 | 笔记量 50-500，A/B象限内容增长 |
| 成熟期 | 标准化流程形成，商业化服务涌现 | 笔记量 >500，C象限教程大量出现 |
| 分化期 | 细分社区出现（按游戏/角色/使用风格） | 出现专属话题标签、专题合集 |

### 5.3 情感态度光谱

将用户对"酒馆替代/补充乙游"的态度映射到光谱上：

```
完全拒绝 ←——→ 好奇观望 ←——→ 尝试使用 ←——→ 双轨并行 ←——→ 完全迁移
   ↑              ↑              ↑              ↑              ↑
 "AI没有灵魂"   "教教我？"    "试了一下       "游戏照玩     "再也不氪了
  "不如原作"    "看起来有趣"    还挺好玩的"    酒馆也用"     酒馆真香"
```

**操作方法：** 对每条笔记和评论标注态度编码，统计各位置的分布密度。

### 5.4 技术门槛-用户画像交叉分析

| | 技术能力较强 | 技术能力一般 | 技术能力较弱 |
|---|---|---|---|
| **重度乙游玩家** | 自行部署酒馆，制作精品角色卡 | 使用云酒馆/移动端方案 | 仅用ChatGPT/DeepSeek直接对话 |
| **轻度乙游玩家** | 可能本身就是技术圈用户 | 跟随教程操作 | 停留在好奇阶段 |
| **泛二次元用户** | 可能已是酒馆老用户 | 被乙游内容吸引入坑 | 通过Character.AI等平台体验 |

## 六、质量控制与注意事项

### 6.1 数据偏差来源

需要在分析中明确标注以下偏差：

- **平台偏差：** 小红书上可见的内容只是冰山一角，大量乙游×酒馆内容流通于QQ群、微博超话、Discord等半私密空间，检索结果必然低估实际规模
- **算法偏差：** 小红书的搜索和推荐算法会影响内容可见性，同一关键词在不同时间、不同账号下的结果可能差异显著
- **自我审查偏差：** NSFW相关内容在小红书上会被主动隐藏或使用隐晦表达，导致这部分使用场景被系统性低估
- **存活偏差：** 被平台删除/限流的内容无法被检索到，而这类内容可能恰好是信号最强的交叉内容

### 6.2 伦理边界

- 不截取或传播用户的私人对话截图
- 不暴露用户在评论区留下的私域联系方式
- 引用笔记内容时做脱敏处理（匿名化处理昵称）
- 对于明显涉及未成年人的内容，仅记录存在性，不做深度分析

### 6.3 效率工具建议

- 使用浏览器插件或脚本批量保存笔记页面（避免内容被删后无法回溯）
- 建立 Excel/Notion 数据库进行结构化记录
- 对高频出现的表达设置文本监测（如通过第三方小红书数据工具追踪关键词热度趋势）
- 可考虑使用千瓜数据、新红数据等小红书分析平台辅助量化分析

## 七、预期产出

完成全部检索与分析后，应能产出以下成果：

1. **内容规模报告：** 小红书上酒馆×乙游交叉内容的总量、增长趋势和活跃度评估
2. **用户画像：** 这一交叉社区的典型用户特征（年龄推断、技术水平、乙游偏好、消费态度）
3. **迁移动机排序：** 驱动乙游玩家尝试酒馆的主要因素及其相对权重
4. **生态地图：** 从小红书可见内容出发，描绘更广泛的酒馆×乙游生态（包括B站、QQ群、微博等跨平台线索）
5. **趋势预判：** 基于内容生命周期分析，判断这一交叉领域的发展方向和潜在拐点
