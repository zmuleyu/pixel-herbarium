# PH v1.1 UI 重设计 Spec

## Context

PH v1.0.0 即将提交 App Store，但截图工作暴露了产品定位与 UI 布局的错位：当前首页以"季节景点信息"为主，核心的拍照+水印功能藏在 Check-in tab 第二层，且水印编辑体验（固定 3×3 网格定位）远低于用户期望。用户明确提出产品定位应转变为**水印工具为主 → 未来叠加 AI 识别**，打卡/收集以"赏花日记"形式呈现。

本 spec 定义 v1.1 的 4 项 UI 重设计：Tab 重组、首页合并、水印编辑器升级、Settings 增强。

**目标版本：** v1.1.0（需 native build，不可 OTA）
**影响范围：** Tab 结构、Home/Footprint/Settings 页面、StampOverlay 手势系统
**设计约束：** Adult Kawaii 调性、PH 配色体系（theme.ts）、日式手帳文化共鸣

## S2: 首页重设计（手帳風 花日記）

### 设计理念

基于日本市场分析：
- **手帳文化** — 目标用户（20-35 都市女性）熟悉「記録 → 回顧 → 共有」模式
- **スタンプラリー心理** — 首页展示已收集作品强化成就感
- **工具优先** — 顶部 CTA 满足水印工具的快速触达

## S3: 水印编辑器升级（自由手势操控）

### 当前问题

1. 3×3 固定网格 — 位置选择不精确，无法微调
2. Slider 调整大小/透明度 — 交互间接，缺乏直观操控感
3. 无旋转 — 水印只能正向放置

### 升级方案

替换 `StampOverlay` + `PositionSelector` + opacity/size sliders 为手势驱动的 `GestureStampOverlay`。

### 新增依赖

```bash
npx expo install react-native-gesture-handler react-native-reanimated
```

**⚠️ 这两个包当前未安装**（package.json 未列出）。虽然 Expo 55 SDK 兼容它们，但必须显式安装。这是 **native 依赖**，安装后必须 native rebuild。

### 前置：GestureHandlerRootView

`react-native-gesture-handler` v2 要求 app 根节点包裹 `<GestureHandlerRootView>`，否则手势在 iOS 静默失效。

**修改 `src/app/_layout.tsx`：**
```typescript
import { GestureHandlerRootView } from 'react-native-gesture-handler';
// 在最外层 View/SafeAreaView 替换为 GestureHandlerRootView
<GestureHandlerRootView style={{ flex: 1 }}>
  <Slot />
</GestureHandlerRootView>
```

### 新组件：`GestureStampOverlay`

```
文件: src/components/stamps/GestureStampOverlay.tsx
```

**手势组合（Simultaneous）：**
- `PanGesture` — 自由拖拽，SharedValue `translateX/Y`
- `PinchGesture` — 双指缩放，SharedValue `scale`（范围 0.3x ~ 2.5x）
- `RotationGesture` — 双指旋转，SharedValue `rotation`（弧度）

**Animated Style：**
```typescript
useAnimatedStyle(() => ({
  transform: [
    { translateX: translateX.value },
    { translateY: translateY.value },
    { scale: scale.value },
    { rotate: `${rotation.value}rad` },
  ],
  opacity: opacity.value,
}));
```

**边界约束：**
- 拖拽范围限制在照片预览区域内（clamp）
- 缩放不超过 2.5x，不小于 0.3x
- 松手后 light elastic 回弹动画（withSpring, damping: 15）

**初始状态：**
- 位置：右下角（与 v1.0 默认 `bottom-right` 一致）
- 缩放：1.0x
- 旋转：0°
- 透明度：从 CustomizationPanel 的効果 tab 控制

### StampPreview 改造

**移除：**
- `PositionSelector`（3×3 网格选择器）
- Opacity slider（20%-100%）
- Size slider（60%-150%）

**保留：**
- `StyleSelector`（6 种样式横滑）
- `CustomizationPanel`（色/効果/文字/飾り）— 新增透明度控制到「効果」tab

**布局重组：**
```
┌─────────────────────────┐
│                         │
│   照片 + GestureStamp   │  ← 预览区域（占屏幕 ~60%）
│   （可拖拽/缩放/旋转）    │
│                         │
├─────────────────────────┤
│ [classic][relief]...    │  ← StyleSelector 横滑
├─────────────────────────┤
│ 🎨色 | ✨効果 | Aa文字 | 🌿飾り │  ← CustomizationPanel tab 化
├─────────────────────────┤
│  [💾 保存]    [↗ 共有]  │  ← 操作按钮
└─────────────────────────┘
```

### captureRef 兼容（⚠️ 需 settle-before-capture 策略）

`react-native-view-shot` 的 `captureRef()` 捕获 native view 状态，而 `useAnimatedStyle` 在 UI 线程运行 worklet。**已知边界情况**：captureRef 可能捕获到手势动画中间帧或初始位置。

**解决方案：settle-before-capture**
```typescript
// 保存前，将 SharedValue 读取到 React state，用静态 style 渲染，再 capture
async function handleSave() {
  // 1. 读取 Reanimated shared values（runOnJS callback）
  const transform = {
    x: translateX.value,
    y: translateY.value,
    scale: scale.value,
    rotation: rotation.value,
  };
  // 2. 设置 React state（触发非 Animated 渲染）
  setStaticTransform(transform);
  setIsCapturing(true);
  // 3. 等一帧确保渲染完成
  await new Promise(r => requestAnimationFrame(r));
  // 4. captureRef
  const uri = await captureRef(viewRef, { format: 'png', quality: 1 });
  setIsCapturing(false);
  return uri;
}
```

`isCapturing` 为 true 时，渲染层切换为普通 `style={{ transform: [...] }}` 而非 `useAnimatedStyle`，确保 captureRef 捕获正确状态。

### 迁移策略

- v1.0 已保存的 `stampPosition` 字段（如 `'bottom-right'`）在 v1.1 中不再使用于编辑器
- 但 `composedUri`（已渲染的图片）不受影响
- CheckinRecord 新增可选字段：`stampTransform?: { x: number; y: number; scale: number; rotation: number }`

## S4: Settings 页面增强

### 新增功能

| 功能 | 优先级 | 说明 |
|------|--------|------|
| 账号区块（登录/资料） | P0 | 未登录时 CTA "ログインする"；已登录显示头像+名前+邮箱 |
| 数据导出 | P1 | 导出打卡记录为 JSON（含 composedUri 路径列表） |
| 数据删除 | P0 | Apple 审核要求，清除本地 Zustand + AsyncStorage |
| 同步状态 | P2 | 登录后显示 Supabase 同步时间戳 |
| 反馈入口 | P1 | 打开邮件 compose 或外部链接 |
| 登出 | P0 | 仅登录后显示，Supabase signOut |
| 账号删除 | P0 | Apple 审核要求，确认对话框 → Edge Function 服务端删除 |

### 账号删除实现（Apple Guideline 5.1.1(v)）

**⚠️ 当前实现不足**：`privacy.tsx` 仅设置 `deletion_requested_at` 字段并 signOut，但未真正删除 Supabase Auth user。Supabase client SDK 无 `auth.admin.deleteUser()` 权限（需 service role key）。

**解决方案：Supabase Edge Function**
```
supabase/functions/delete-account/index.ts
→ 验证 JWT → supabaseAdmin.auth.admin.deleteUser(userId) → 级联删除 profiles row
→ 返回 { success: true }
```

客户端调用：
```typescript
const { error } = await supabase.functions.invoke('delete-account');
if (!error) { signOut(); clearLocalData(); }
```

### 数据导出范围

**导出内容**：本地 `checkin-store` 的 history 数组（JSON 格式）：
- 每条记录：`{ spotId, spotName, timestamp, stampStyle, composedUri(本地路径) }`
- **不含** Supabase 远端数据（discoveries 表由 `privacy.tsx` 已有导出逻辑处理）
- **复用** `src/app/privacy.tsx` 中已有的导出 UI 模式，避免重复实现

## 非目标 / 约束

- **不做** AI 识别功能（v2 scope，代码已有但 hidden）
- **不做** 社交/好友系统（v2 scope，tab hidden）
- **不做** 地图 tab（v2 scope，tab hidden）
- **不做** 自定义图标设计（预留空间，后续专项优化）
- **不做** 水印模板市场/付费功能（YAGNI）
- **必须** native build（新增 gesture-handler + reanimated native 依赖）
- **清理** `@react-native-community/slider` + `react-native-svg` — 如 size slider 被移除且 StampDecoration 不再需要独立 SVG 包，可考虑移除（需验证其他用途）
- **必须** 保持 Adult Kawaii 调性（温柔文案、不制造焦虑）
- **必须** 使用 theme.ts 颜色常量
