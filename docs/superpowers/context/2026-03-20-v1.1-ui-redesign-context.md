# Context: PH v1.1 UI Redesign

**What we're building:** Restructure PH from a "flower spot check-in app" to a "photo watermark tool + flower diary", with 3 tabs (Home/Check-in/Settings), gesture-based stamp editing, and enhanced Settings with account management.

**Locked decisions:**
- 4→3 visible tabs: Home (diary + CTA) | Check-in (watermark editor) | Settings (account + data)
- Footprint tab set to `visible: false` (not deleted — OTA safety contract)
- Stamp positioning: free drag + pinch zoom + rotation (replacing 3×3 grid)
- Uses `react-native-gesture-handler` + `react-native-reanimated` (must install, not currently in package.json)
- `GestureHandlerRootView` wraps app root in `_layout.tsx`
- captureRef uses settle-before-capture strategy (SharedValue → React state → static render → capture)
- Account deletion via Supabase Edge Function (not client-side SDK)
- Data export: local checkin-store only; Supabase data uses existing privacy.tsx
- Must be native build (not OTA) due to new native deps + tab structure change

**Non-goals / constraints:**
- No AI identification (v2, code preserved but hidden)
- No social/map/profile tabs (v2, hidden)
- No custom icon design (deferred, space reserved)
- No watermark marketplace/IAP (YAGNI)
- Must maintain Adult Kawaii tone (no anxiety-inducing copy)

**Resolved edge cases:**
- 3 navigation references to `/(tabs)/footprint` → updated to `/(tabs)/home` (checkin.tsx:230, _layout.tsx:66, demo-data.ts)
- VoiceOver/TalkBack users → fallback to 3×3 grid when reduceMotion is on
- Existing CheckinRecord `stampPosition` field → ignored in v1.1 editor, `composedUri` unaffected
- `@react-native-community/slider` may become unused → clean up in T5
