## 1. 设计定位与核心原则

### 1.1 水印的本质定义

水印在本产品中不是「功能标注」，而是**情感凭证**——它是用户与某处樱花相遇的有形证明。这个定义决定了所有设计取舍：

- 水印应融入照片，而非凌驾于照片之上
- 水印应传递「我在这里」的仪式感，而非广告感
- 水印信息应有内容密度，但排版需保持克制

### 1.2 四项核心设计原则

**① 情感优先于功能**
编辑器的每个交互节点都应强化「这次相遇值得记录」的情感基调。确认弹窗用「遇见」而非「打卡」，按钮文案用「分享这次相遇」而非「导出图片」。

**② 默认值即最优解**
用户打开编辑器时，水印的风格、位置、透明度、大小均已由系统预设为最优方案。大多数用户不需要调整任何参数，直接点击分享即可完成完整体验。

**③ 先验证，后编辑**
GPS 位置验证必须发生在照片编辑之前。顺序反转会导致两个问题：一是用户可能用历史照片作弊，二是「当场纪念」的仪式感消失，水印失去意义。

**④ 错开反馈，避免信息过载**
打卡成功后有多个正向反馈节点（分享成功提示 → 图鉴解锁动效 → 进度里程碑通知）。三者应错开出现，间隔不低于 800ms，避免用户被同时涌入的反馈淹没。

## 2. 完整用户旅程

```
[发现] 地图页 → 附近满开预报推送 / 主动搜索地点
   ↓
[导航] 点击地点卡片 → 查看品种信息、花期预报、其他用户打卡记录
   ↓
[接近] GPS 检测用户进入地点半径（默认 100m）
   ↓
[确认] 「到访确认」浮层弹出 → 用户手动确认（防误触）
   ↓
[取材] 选择路径：
       ├── 即时拍摄（调用系统相机）
       └── 从相册选图（近 30 天内照片，防止用旧图）
   ↓
[编辑] 进入水印编辑器
       ├── 系统自动加载该地点的水印信息
       ├── 智能检测照片主体，预设最优水印位置
       ├── 用户可调整：风格 / 位置 / 透明度 / 大小
       └── 实时预览，所见即所得
   ↓
[导出] 点击「分享这次相遇」
       ├── 选择目标平台（LINE / Instagram / 保存到相册 / 更多）
       └── 导出带水印的高分辨率图片
   ↓
[反馈-1] 分享成功提示（0ms）：轻量 Toast，持续 2s
   ↓
[反馈-2] 图鉴解锁动效（800ms 后）：像素樱花翻页动画
   ↓
[反馈-3] 里程碑提示（按需，图鉴解锁后 1s）：如「已收集 5 处！」
   ↓
[沉淀] 图鉴页新增一页，照片永久归档，可随时回顾
```

## 3. 到访确认模块

### 3.1 触发条件

| 条件 | 参数 | 说明 |
|------|------|------|
| 主要触发 | 用户进入地点半径 100m | GPS 持续检测，误差 ±15m |
| 辅助触发 | 用户在半径内停留 ≥ 30s | 避免路过误触发 |
| 冷却机制 | 同一用户同一地点，24h 内仅弹出一次 | 防止反复提醒骚扰 |
| 手动触发 | 用户在地点详情页主动点击「我在这里」 | 覆盖 GPS 精度不足场景 |

### 3.2 弹窗设计规范

**弹窗结构（从上至下）**

```
┌──────────────────────────────────┐
│  [地点像素图鉴缩略图，72×72]       │
│                                  │
│  你到了                           │  ← 主标题，16px，字重 500
│  哲学之道                         │  ← 地点名，22px，字重 600，品牌粉色
│                                  │
│  関山桜 · 京都市左京区             │  ← 副信息，13px，静音色
│  今日满开预报 ●                   │  ← 状态标签，满开用粉色，未满开用灰色
│                                  │
│  ┌─────────────────────────────┐  │
│  │    📷  为这次相遇留一页      │  │  ← 主操作按钮，品牌粉底白字
│  └─────────────────────────────┘  │
│                                  │
│  不是现在                         │  ← 次级操作，13px，静音色，无下划线
└──────────────────────────────────┘
```

**设计细节**

- 弹窗从底部滑入（Bottom Sheet），高度约屏幕 50%，背后有轻度毛玻璃遮罩
- 地点缩略图展示该地点的像素图鉴风格图，即将解锁的「奖励预览」，激发期待感
- 「不是现在」使用文字而非按钮，降低视觉权重，引导用户选择主操作
- 点击遮罩区域等同于「不是现在」，不触发打卡流程

### 3.3 照片来源选择

确认到访后，呈现照片来源选择：

```
┌────────────┬────────────┐
│  📷        │  🖼️        │
│  现在拍摄   │  从相册选择  │
│            │  (近30天)   │
└────────────┴────────────┘
```

**相册限制说明**：照片筛选条件为「拍摄时间在近 30 天内」，而非「今日」，兼顾用户在同一地点多日游览的场景，同时防止用数年前的旧照片打卡。若用户选择的照片超出时间范围，系统给予温和提示而非硬性阻断：

> 「这张照片拍于 XX 天前，确定用它记录今天的相遇吗？」
> [使用这张] [重新选择]

## 4. 水印编辑器——核心交互设计

### 4.1 界面布局结构

```
┌─────────────────────────────┐
│  顶部导航栏                  │  ← 44px，含返回、地点标签、完成
├─────────────────────────────┤
│                             │
│                             │
│       照片预览区              │  ← 占屏幕约 55%，4:3 或 9:16 自适应
│                             │  ← 水印实时叠加预览
│              [水印]          │
├─────────────────────────────┤
│  风格选择 Tabs               │  ← 像素 / 印章 / 简约，3列等宽
│─────────────────────────────│
│  位置 9宫格 + 缩略图预览      │
│─────────────────────────────│
│  透明度滑块                  │
│  大小滑块                    │
├─────────────────────────────┤
│  [分享这次相遇 →]             │  ← 主操作按钮，固定在底部
└─────────────────────────────┘
```

### 4.2 自动初始化逻辑

编辑器打开时，系统自动完成以下初始化，用户无需手动设置：

**① 水印内容填充**（来源于已验证的打卡地点数据）
- 地点名称（日语 + 中文）
- 樱花品种名（日语学名）
- 花言葉
- 当前日期（`YYYY.MM.DD` 格式）
- 城市名称（英文）
- 品牌标识

**② 风格预选**（基于用户历史偏好）
- 新用户默认选「像素」风格（信息量最大，适合第一次体验）
- 回访用户沿用上次选择的风格

**③ 智能位置推荐**（基于图像分析）

| 照片类型 | 默认水印位置 | 逻辑说明 |
|---------|------------|---------|
| 主体居中（如人像） | 右下角 | 避免遮挡主体 |
| 主体偏左 | 右下角 | 利用右侧留白 |
| 主体偏右 | 左下角 | 利用左侧留白 |
| 满构图（花海） | 右下角 | 通用最优默认 |
| 竖版九比十六 | 左下角 | 避开 Instagram 安全区域 |

**④ 透明度默认值**：`90%`（清晰可见但不刺眼）

**⑤ 大小默认值**：`100%`（基准尺寸，设计时已考虑与不同分辨率的比例协调）

### 4.3 实时预览要求

所有参数调整（风格切换、位置移动、透明度、大小）的视觉反馈必须在 **100ms 以内**呈现，确保用户感受到即时响应。

位置移动使用 CSS `transition: all 0.28s ease`，形成平滑的位移动效，而非瞬间跳跃。大小缩放使用 `transform: scale()` 而非改变实际尺寸，保持布局稳定。

## 5. 三种水印风格规范

### 5.1 风格定位矩阵

| 维度 | 像素（Pixel） | 印章（Seal） | 简约（Minimal） |
|------|-------------|------------|--------------|
| **视觉重量** | 重 | 中 | 轻 |
| **信息密度** | 高（5项信息） | 中（3项信息） | 低（2项信息） |
| **适用平台** | 小红书、APP内分享 | Instagram、纸质打印 | 朋友圈、故事 |
| **用户心态** | 「晒出收集进度」 | 「留存纪念品」 | 「照片主体不被干扰」 |
| **品牌辨识度** | 强 | 中 | 弱（刻意降低） |

### 5.2 像素风格（Pixel）规范

**视觉特征**：矩形边框，像素感排版，带品牌 Logo 和完整地点信息

```
┌──────────────────────┐  ← 边框：2px solid，品牌粉 #D4537E
│ ✿ PIXEL HERBARIUM    │  ← 图标 + 品牌名，8px，monospace，字间距 1px
│                      │
│ 哲学之道              │  ← 地点名，17px，衬线字体，字重 700，#D4537E
│                      │
│ 2026.03.28 · KYOTO   │  ← 日期+城市，8px，monospace，#A05070
└──────────────────────┘
  背景：白色 rgba(255,255,255,0.93)
```

**内容字段优先级**（空间不足时按序省略）：
1. 地点名（必须保留）
2. 品牌标识（必须保留）
3. 日期
4. 城市名
5. 花言葉（可选，仅在「展开版」中显示）

### 5.3 印章风格（Seal）规范

**视觉特征**：圆形构图，仿印鉴质感，适合叠加在照片边角

```
      ╭─────────╮
   ╭──╯         ╰──╮
  │   哲学之道      │  ← 地点名，8px，字间距 2px
  │    ✿  ✿  ✿    │  ← 樱花像素图标，20px
  │    2026春      │  ← 年份+季节，7px，字间距 1.5px
   ╰──╮         ╭──╯
      ╰─────────╯
```

- 直径：74px（默认 100% 大小时）
- 边框：2.5px solid #D4537E
- 背景：rgba(255,255,255,0.90)
- 内部文字围绕圆心排列，产生「古典印鉴」联想

### 5.4 简约风格（Minimal）规范

**视觉特征**：左侧色边 + 深色背景文字块，极简，不打断视线

```
║ 哲学之道            ← 地点名，13px，字重 600，白色 rgba(255,255,255,0.97)
║ Kyoto · 2026.03.28  ← 副信息，9px，白色 rgba(255,255,255,0.78)
```

- 左侧色边：2.5px solid #ED93B1（浅粉，比 Pixel 主色调更轻）
- 背景：rgba(0,0,0,0.32)（半透明深色，与任何照片底色兼容）
- 内边距：5px 8px
- 信息仅保留地点名 + 地点副信息，品牌 Logo 不出现

## 6. 位置选择系统

### 6.1 9宫格控件设计

**控件由三部分组成**：

```
[位置标签]  [9格宫格]  [实时缩略预览]
```

9格宫格的每个格子代表照片中的一个区域，点击即激活对应位置。激活状态：格子背景变为浅粉 `#FBEAF0`，边框变为品牌粉 `1.5px solid #D4537E`，中心点变为品牌粉实心圆。

**实时缩略预览**（控件右侧的小卡片）：一个 40×36px 的照片缩略图，内有一个 5×5px 的粉色方块实时跟随宫格位置移动。用户不需要抬头看大预览区就能感知水印将落在哪里，降低认知摩擦。

### 6.2 位置坐标映射

| 宫格编号 | 水印锚点 | CSS 实现 |
|---------|---------|---------|
| 0（左上） | top: 16px, left: 16px | `top: 16px; left: 16px` |
| 1（上中） | top: 16px, 水平居中 | `top: 16px; left: 50%; transform: translateX(-50%)` |
| 2（右上） | top: 16px, right: 16px | `top: 16px; right: 16px` |
| 3（左中） | 垂直居中, left: 16px | `top: 50%; left: 16px; transform: translateY(-50%)` |
| 4（正中） | 水平+垂直居中 | `top: 50%; left: 50%; transform: translate(-50%,-50%)` |
| 5（右中） | 垂直居中, right: 16px | `top: 50%; right: 16px; transform: translateY(-50%)` |
| 6（左下） | bottom: 16px, left: 16px | `bottom: 16px; left: 16px` |
| 7（下中） | bottom: 16px, 水平居中 | `bottom: 16px; left: 50%; transform: translateX(-50%)` |
| 8（右下）✦ | bottom: 16px, right: 16px | `bottom: 16px; right: 16px` |

✦ 系统默认位置

### 6.3 安全区域检测

部分位置在特定照片构图下不适合放置水印。系统应检测以下情况并给予提示（非阻断）：

- **4号（正中）**：若照片主体在中心区域，以淡黄色背景提示「水印可能遮挡照片主体」
- **0/1/2号（顶部）**：若 APP 界面将在顶部叠加状态栏，提示「分享后顶部水印可能与状态栏重叠」
- **社交平台安全区**：用户选择 Instagram 为分享目标时，自动检测是否有位置落在 Instagram Story 的点击区域内，如有则建议切换

## 7. 参数调节交互

### 7.1 透明度滑块

| 参数 | 值 |
|------|---|
| 最小值 | 20%（完全隐藏水印无意义，最低保留轮廓） |
| 最大值 | 100% |
| 默认值 | 90% |
| 步长 | 1% |
| 右侧数值显示 | 实时更新，格式：`93%` |

**特殊节点触觉反馈**（Haptic Feedback）：当滑块经过 50%（半透明）和 100%（完全不透明）时，触发轻微震动反馈（仅 iOS / 支持震动的 Android 设备）。

### 7.2 大小滑块

| 参数 | 值 |
|------|---|
| 最小值 | 60%（过小会导致文字难以辨认） |
| 最大值 | 150%（超过此值水印喧宾夺主） |
| 默认值 | 100% |
| 步长 | 1% |

**缩放锚点规则**：缩放以水印的当前位置锚点为中心展开，而非始终以某一固定角为基准。例如水印在右下角时，缩放以右下角为原点（`transform-origin: right bottom`），确保缩放不会导致水印偏移出预期区域。

### 7.3 双指手势支持（进阶交互）

在照片预览区，支持用户直接用手势操作水印（类似贴纸编辑体验）：

- **双指捏合/展开**：调整大小（同步更新大小滑块的数值）
- **单指拖拽水印**：自由移动位置（落点吸附到最近的宫格位置，同步更新宫格选中状态）
- **双指旋转**（仅印章风格支持，V2 版本上线）

吸附逻辑：当水印中心点距离任一宫格区域中心 ≤ 30px 时，自动吸附并触发轻微震动反馈。

## 8. 分享流程与多平台适配

### 8.1 分享平台选择界面

点击「分享这次相遇」后，弹出底部平台选择面板：

```
┌─────────────────────────────────────┐
│  选择分享方式                         │
│                                     │
│  [LINE]  [Instagram]  [X]  [微信]    │  ← 第一行：主推平台（基于用户偏好排序）
│                                     │
│  [保存到相册]  [更多...]              │  ← 第二行：本地保存和系统分享
└─────────────────────────────────────┘
```

**平台排序逻辑**：基于用户过往分享记录的平台频次排序，新用户默认顺序为「LINE > 保存到相册 > Instagram > X」（依据日本市场用户习惯）。

### 8.2 各平台图片规格适配

| 平台 | 推荐尺寸 | 格式 | 特殊处理 |
|------|---------|------|---------|
| LINE | 1080×1080 | JPG 90% | 正方形裁切，水印相对尺寸需重新计算 |
| Instagram Feed | 1080×1350 | JPG 90% | 4:5 竖向，预留底部品牌区域 |
| Instagram Story | 1080×1920 | JPG 90% | 9:16，顶部 250px 为点击区避开 |
| X（Twitter） | 1200×675 | JPG 85% | 16:9 横向，水印建议移至右下角 |
| 保存到相册 | 原始分辨率 | JPG 95% | 保持最高质量 |

**尺寸适配提示**：当用户选择的平台与当前照片比例不一致时，提示用户可选择裁切方式（居中裁切 / 手动调整），并预览水印在新比例下的位置。

### 8.3 照片导出质量策略

- 水印以**矢量方式**叠加于原图之上，而非光栅化后压缩，确保水印在任何分辨率下均清晰
- 像素风格水印保留其像素感（不进行抗锯齿平滑），与品牌视觉一致
- 导出过程显示进度条，预计时间 < 2s（基于 4200 万像素手机相机原图）

## 9. 图鉴解锁联动设计

### 9.1 时机与触发条件

图鉴解锁动效**不在用户点击「分享」时立即触发**，而是：

```
用户点击分享 → [0ms] 分享成功 Toast 出现
             → [800ms] 分享 Toast 消失
             → [1000ms] 图鉴解锁动效开始
             → [2500ms] 动效结束，图鉴新页面静止展示
             → [3500ms] 如触发里程碑，里程碑提示出现
```

错开 800ms 的原因：让用户有时间感受「分享成功」这个节点，再接收「图鉴解锁」这个新的情感高潮，避免两个正向激励相互稀释。

**特殊情况**：如果用户选择「保存到相册」而非分享到社交平台，图鉴解锁仍然正常触发（打卡行为本身已经完成，不依赖实际分享）。

### 9.2 图鉴解锁动效分解

**阶段 1（0–400ms）**：像素樱花花瓣从水印位置生成，向四周散开，带有轻微物理感的抛物线轨迹

**阶段 2（400–1200ms）**：散开的花瓣向屏幕中心汇聚，形成一本翻开的像素图鉴书

**阶段 3（1200–2000ms）**：图鉴书翻到新的一页，该地点的完整图鉴页面从右侧滑入并定格，展示：
- 地点大图（像素插画风格）
- 樱花品种名 + 花言葉
- 用户刚拍的照片缩略图（作为「亲历记录」附在图鉴角落）
- 打卡日期

**阶段 4（2000–2500ms）**：图鉴页面渐入稳定，底部出现「查看完整图鉴」按钮

### 9.3 再次到访的差异化动效

当用户再次打卡同一地点时（非首次），动效有所区别：

- 不显示翻页动效（该页已存在）
- 改为在已有图鉴页上「添加一条日期记录」的效果，小印章落在角落
- 提示文案：「欢迎回来。去年你也来过这里。」

## 10. 动效规范

### 10.1 基础动效参数

| 动效类型 | 时长 | 缓动曲线 | 说明 |
|---------|------|---------|------|
| 水印位置移动 | 280ms | ease | 平滑位移，不过冲 |
| 大小缩放 | 200ms | ease | 轻快缩放 |
| 透明度变化 | 150ms | linear | 线性渐变，直觉最符合 |
| 风格切换 | 200ms | ease-in-out | 旧风格淡出，新风格淡入 |
| Bottom Sheet 弹出 | 380ms | cubic-bezier(0.25, 0.1, 0.25, 1) | 标准 Material 曲线 |
| 按钮点击反馈 | 80ms | ease-out | 按下缩放至 96%，松开弹回 |

### 10.2 花瓣散射动效参数（图鉴解锁）

- 花瓣数量：8–12 片（随机）
- 初始位置：水印所在位置的中心点
- 运动轨迹：抛物线（初始速度随机，重力加速度统一）
- 旋转：每片花瓣旋转速度随机，范围 60–240 deg/s
- 颜色：在 `#F4C0D1`（浅粉）到 `#D4537E`（深粉）之间随机
- 消散：落点后淡出，持续 200ms

### 10.3 减少动态效果偏好支持

遵循 `prefers-reduced-motion` 媒体查询：
- 检测到用户开启「减少动态效果」时，所有动画替换为即时状态切换（0ms transition）
- 图鉴解锁动效替换为简单的淡入淡出（300ms opacity）

## 11. 异常状态处理

### 11.1 GPS 信号异常

| 场景 | 处理方式 |
|------|---------|
| GPS 定位失败 | 提示用户手动确认位置，展示地点地图截图供对照 |
| GPS 精度不足（误差 >50m） | 正常允许打卡，但在图鉴页标注「位置精度较低」（用户不感知） |
| GPS 显示用户不在附近（>500m） | 柔和提示「你似乎还没到达那里，确定要打卡吗？」给予用户确认选项，不强制阻断 |

**设计原则**：不因技术误差惩罚真实前往的用户。宁可接受少量异常打卡，也不因误判而让真实到访的用户感到沮丧。

### 11.2 照片相关异常

| 场景 | 处理方式 |
|------|---------|
| 照片超出 30 天范围 | 温和提示，允许用户强制使用（保留用户自主权） |
| 照片分辨率过低（<720p） | 提示「图片质量较低，导出后可能模糊」，继续允许操作 |
| 照片包含 EXIF 位置信息与 GPS 不符 | 不作任何提示（不过度侦查用户隐私） |
| 相机权限被拒绝 | 引导至系统设置授权，提供相册选图作为替代方案 |

### 11.3 分享异常

| 场景 | 处理方式 |
|------|---------|
| 分享被目标 APP 取消 | 不重置编辑器，用户可重新选择平台或直接保存到相册 |
| 网络异常导致分享失败 | 先保存到相册，提示用户稍后手动分享 |
| 导出超时（>5s） | 展示进度条，超过 5s 提示「处理中，请稍候」，不终止任务 |

即便分享失败，**图鉴解锁仍然正常完成**。用户完成了到访验证，图鉴页应当得到更新，分享是独立的下游操作。

## 12. 无障碍设计

### 12.1 VoiceOver / TalkBack 支持

- 所有交互元素提供 `accessibilityLabel`，描述操作而非外观
  - 宫格格子：「选择右下角位置」而非「格子 8」
  - 风格按钮：「像素风格，当前选中」
  - 滑块：「透明度，当前 90%」

- 图片预览区提供描述：「哲学之道樱花照片，当前水印位于右下角，像素风格」

### 12.2 色彩对比度

- 所有文字元素（包括水印内文字）与背景对比度 ≥ 4.5:1（AA 级）
- 像素风格的白底粉框在深色照片上可能对比度不足：自动检测照片该区域的平均亮度，如背景过暗则自动为白底添加淡灰色阴影边框

### 12.3 最小触控区域

- 所有可点击元素（宫格格子、Tab 按钮、操作按钮）的实际触控响应区域 ≥ 44×44px，即便视觉尺寸更小

## 13. 数据埋点建议

### 13.1 关键漏斗事件

```
checkin_prompt_shown        → 到访确认弹窗出现
checkin_confirmed           → 用户点击「为这次相遇留一页」
photo_source_selected       → 拍照 / 相册（带来源字段）
editor_opened               → 编辑器成功初始化
style_changed               → 风格切换（带 from / to 字段）
position_changed            → 位置调整（带格子编号字段）
opacity_adjusted            → 透明度调整（带最终值）
size_adjusted               → 大小调整（带最终值）
share_button_tapped         → 点击分享按钮
share_platform_selected     → 选择分享平台（带平台字段）
share_completed             → 分享成功
share_failed                → 分享失败（带失败原因）
herbarium_unlocked          → 图鉴解锁完成
milestone_reached           → 里程碑达成（带里程碑类型）
```

### 13.2 核心分析维度

- **风格偏好分布**：监测三种风格的选择比例，指导后续风格迭代优先级
- **位置偏好热力图**：9个位置的选择频次分布，验证默认位置（右下角）的合理性
- **参数调整率**：多少用户在默认值基础上进行了调整，高调整率意味着默认值需要优化
- **分享完成率**：进入编辑器 → 分享成功的转化率，低于 70% 需排查流程障碍
- **图鉴解锁率**：分享完成 → 图鉴解锁的比率，应接近 100%，异常则排查技术问题
