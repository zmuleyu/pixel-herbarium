# App Store 截图方案

> 参考：NxCode 自动生成截图建议 + 上传到 App Store Connect 的思路。
> 目标：5-8 张高质量截图，突出 PH 核心差异化。

---

## 截图尺寸规格

| 设备 | 分辨率 | App Store 要求 |
|------|--------|--------------|
| **iPhone 16 Pro Max (6.9")** | **1320 × 2868 px** | ✅ 必须 |
| iPhone 14 Plus (6.5") | 1284 × 2778 px | 推荐（6.9" 无法自动缩放时需要）|
| iPhone 8 Plus (5.5") | 1242 × 2208 px | 可选（6.5" 会自动缩放填充）|

> **最少要求：** 只需上传 6.9" 截图，App Store Connect 会自动缩放到其他尺寸。
> 但建议同时准备 6.5" 以保证在老设备上的显示质量。

## 拍摄方案

### Option A：真机拍摄（推荐）

**前提：** 手机已安装 preview build（最新 `40c5a289`）

```
1. 在 iPhone 上运行 App（preview build，非 Expo Go）
2. 准备好各截图对应的界面状态（提前造数据：收集几种植物、翻几张卡）
3. 截图快捷键：侧边按钮 + 音量上键（iPhone X+）
4. 截图保存到相册后传到电脑
```

**优势：** 真实感强，审核员也会看到真实渲染效果

### Option B：Simulator（需要 Mac）

```
1. Xcode → Simulator → iPhone 16 Pro Max
2. 加载 OTA 或直接 npx expo start（Simulator 模式）
3. Device → Screenshot (Command+S) → 保存到桌面
```

**优势：** 分辨率精确，截图一致性高

## 后期处理（添加设备边框 + 文字叠加）

### 工具选项

| 工具 | 特点 | 建议用途 |
|------|------|---------|
| **Canva** | 有 iPhone 截图模板，拖拽操作 | 快速出图 |
| **Figma** | 精确控制，品牌一致性高 | 需要像素级调整时 |
| **AppScreens.io** | App Store 截图专用工具 | 批量多尺寸生成 |
| **手动（无边框）** | 直接上传截图，无设备框 | App Store 允许，但边框更专业 |

### 文字叠加规范（参考 PH 设计语言）

```
字体：日文 - 游ゴシック Medium / 英文 - SF Pro Display
背景：#f5f4f1（PH 主色）或截图画面延伸
文字颜色：深褐 #3d2c1e（主标题）/ 暖灰 #7a6b5a（副标题）
```

---

## 注意事项

- **不要在截图中展示未实现的功能**（如 LINE 花束如果 v1.0 未上线）
- **不要在截图中显示价格**（价格在各地区不同，App Store 不允许）
- **状态栏要干净**：时间设为 9:41（Apple 惯例），无低电量提示
- **截图不能有 Home Indicator 条**（或统一保留）

---

## 上传流程

```
App Store Connect → 花図鉑 → iOS App → 1.0 准备提交 → App 预览和截图
→ 选择 "iPhone 6.9 英寸" → 上传 PNG 文件（最多 10 张）
→ 拖拽排序（第 1 张最重要，代表搜索结果主图）
```
