# PH 运营监控指标基线

> 上线后建立 · 版本 v1.0 · 2026-03-20

---

## 核心指标（优先级 P0）

| 指标 | 目标值 | 告警阈值 | 数据源 |
|------|--------|---------|--------|
| 崩溃率 | < 1% | > 2% | Sentry / EAS |
| D1 留存 | > 40% | < 25% | Supabase analytics |
| DAU | 增长期 > 10%/周 | 连续下降 3天 | Supabase |
| App Store 评分 | ≥ 4.5 | < 4.0 | App Store Connect |
| 评论数 | 前30天 > 10 | — | App Store Connect |
| 关键词排名均值 | 前10名 | 跌出前20 | ASO工具 |

---

## 技术健康指标（P1）

| 指标 | 目标值 | 告警阈值 |
|------|--------|---------|
| API 错误率 | < 0.5% | > 2% |
| OTA 更新成功率 | > 95% | < 90% |
| Supabase 响应时间 | < 200ms p95 | > 500ms |
| Edge Function 错误 | < 1% | > 5% |

---

## 季节性内容指标（P1）

| 指标 | 说明 |
|------|------|
| 打卡转化率 | 打开 → 打卡 / 总打开（目标 > 15%） |
| 每用户打卡次数/季节 | 赏花季应 > 2 次/人 |
| 花図鉑收集深度 | 用户平均解锁植物数（目标 > 10 种） |
| 分享率 | 有分享行为的用户占比（目标 > 5%） |

---

## 监控工具配置

### Sentry（崩溃监控）
- 项目：`pixel-herbarium-ios`
- 告警规则：崩溃率 > 2% → 邮件通知
- 阈值更新时机：上线后第一周数据稳定后设定基线

### App Store Connect
- 通过 `scrapling-mcp` 的 `appstore_reviews` 工具定期采集
- `ph-daily` cron 任务自动更新 `app-status.json` metrics 字段

### Supabase Analytics
- `analytics_events` 表记录核心行为事件
- 每周从 Supabase Dashboard 导出关键漏斗数据

---

## 告警响应 SOP

1. 崩溃率 > 2%：立即检查 Sentry → 评估是否 OTA 可修复 → 24h 内发布修复
2. 评分 < 4.0：查看最新评论 → 识别共性问题 → 优先修复后回复评论
3. DAU 连续下降 3天：检查是否有竞品大版本发布 / 季节转换 → 推送激活

---

*上线前：所有指标均为 null（见 app-status.json metrics 字段）*
*上线后：前7天数据建立基线，第7天更新本文件*
